Note: When representing the world with geographic information the order of layers is important. Software for managing geographic information has some similarities with video and image editing software such as Photoshop or Premier Pro. The order that you arrange your layers will determine how we see them.
It is very important that you understand how to save your files in an organized manner: When you save a project in QGIS (or ARCGIS or Premier Pro or any other software that deals with big datasets) you are not necessarily saving all of the data inside that project. The project only has the instructions on what to do with your data. If you move or rename your files, the project might have problems opening your map. So don't forget to keep your projects organized!

4. Maptile layer

Required video
Tutorial video 1e on adding a tile layer

By the end of this section you should have a general idea of how map tiles look, and how to add them as a layer in QGIS.
Traditionally, geovisualizations were done by working with vector and raster data opened from within your local computer environment or network. Today, the most common way that we consume maps is through our phones, and the data comes from the internet. Tiling images is a very effective way of transfering spatially explicit visual data through the internet. For example, the visual component of Google Maps or Open Street Maps are images that have been tiled. By the end of the semester you will learn how to make an original tile map. But for now, you will learn how open tiles in QGIS.
You are not expected to understand the math or the technical details on how to make a tile layer, however if you are interested in this kind of stuff you can look at this page on OSM tiles or this page on Bing Map Tiles, which has a nice table on the level of detail for each zoom.

By the end of this section you should be able to create your own vector data. You will be able to digitize your original points, lines and polygons.
For this introductory class, we are only doing simple "spaghetti model" digitization of vectors. However, if you are serious about digitizing, or have a large project (eg: roads, land cover map) it is important to learn GIS topology rules. Creating a clean vector topology dataset in GIS is something which I find much more straightforward to do in ESRI than in QGIS. However, new plugins have been created for QGIS (almost every month) that I have not tested, so it may become easier in QGIS.

6. Gain experience using GIS file formats

Required reading

This is a (pretty incomplete) list of GIS file fomats from Wikipedia but it can give you an idea of the plethora of raster and vector formats out there

By the end of this section you should know that:
- Vector and raster files come in different formats.
- Files with extensions like .shp, .geoJSON, .kml are common vector formats.
- Files with extensions like .tif, .png and .img are some examples of raster formats.

The .shp format is still the most common spatial data format out there for vectors, but it has many limitations, as discussed in Switch from Shapefile. The geoJSON format is the most common webmap vector format. One of the benefits of geoJSON is that it can be read by any text editor. However, that could also be a limitation when compared to a shapefile: for example, a geoJSON with the same amount of information as a shapefile will often be larger in size. A benefit of geoJSON is that GitHub can natively display a geoJSON over a map.
