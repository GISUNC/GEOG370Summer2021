2. Quering a dataset using boolean operators

Required reading:
Searches and Queries, pages 144-149. The book describes how queries are done in SQL. However, in class we will be using QGIS Query Builder, which uses an SQL-like WHERE clause.
QGIS information on queries

Required videos:
Required data for the next video
Tutorial video_4d on selecting features using UNC buildings and bike racks layers
Tutorial video_4e on making a query using the UNC bike rack dataset

By the end of this section you should be able to:
- Identify what is a field (column) and what is a feature (row).
- Identify common data types (integer, decimal, string and date).
- Select features manually and export them in a different format and projection.
- Make a query to search for data within a range of values or for an specific value.
- Use QGIS Query Builder format, using double quotes ("Cities") for the field name, single quotes ('Chapel Hill') for text, and no quotes for numbers.
- Make queries that contain boolean AND and OR operators.

- Apply simple techniques to clean your data before uploading the data to QGIS, thus facilitating the process of joining your data
- Obtain Census data and make choropleths related to topics of your interest
